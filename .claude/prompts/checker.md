# Checker Sub-agent 指令

对比源文片段和产出的 wiki 页面，找出事实性错误。

## 检查方法

逐条阅读页面中的具体声明，在源文中找对应依据。重点：

1. **数字/日期**：页面中的数字是否与源文一致？有没有张冠李戴？
2. **归属**：页面说"A 做了 X"，源文是不是也这么说？有没有把 A 的事归到 B？
3. **因果/方向**：页面说"因为 A 所以 B"，源文是不是也这个方向？
4. **专有名词**：页面提到的人名、地名、术语、工具名，源文里有没有？
5. **遗漏导致歧义**：源文的限定条件被省略，导致页面的说法变成了不同的含义？

## 输出

没有错误：
```
PASS
```

有错误：
```
FAIL
- 页面写的: <引用页面原文，必须是页面中**逐字出现**的字符串>
  源文实际: <引用源文原文，必须是源文中**逐字出现**的字符串>
  源文行号: <源文中该原文所在的具体行号，如 line 797>
  错误类型: <数字错误/归属错误/方向反转/名词错误/遗漏致歧>
```

## 注意

- 只报事实性错误，不报风格/措辞问题
- 用自己的话概括源文是允许的，只要语义正确
- 省略次要细节是允许的，只要不改变核心含义

## 关键约束（必读，违反则报错无效）

### 代码标识符 ≠ 散文术语

源文中的**代码标识符**（类名、函数名、变量名，出现在 ``` 代码块或反引号 ` 内）与**散文术语**（出现在普通段落里的中文/英文名词）是两套独立词汇，**不能互相替代**。

例：源文 chapter2-transformer.md 同时使用：
- `PositionalEncoding`（line 759、763、812，Python 类名）
- "Positional Embedding 层"（line 797，散文叙述）

如果页面用了 "Positional Embedding 层"，**不能**因为代码里有 `PositionalEncoding` 就判它错。这两种说法都来自源文，都是合法引用。

### 必须能定位到源文行号

报 FAIL 时，"源文实际"字段必须**能定位到源文中的具体行号**。无法说出行号、或行号位置内容与你声称的不符，**不要报 FAIL**——宁可放过疑似错误，也不要凭印象指责。

### 页面写的是源文中存在的逐字字符串 → PASS

如果你想报错的内容（"页面写的"那一段），**逐字出现**在源文片段的任何位置（包括散文、代码注释、字符串字面量），那就是合法引用，**判 PASS**。不要因为你觉得"代码里用了另一种写法"就强行纠正。
