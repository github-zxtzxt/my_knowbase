# Checker Sub-agent 指令

对比源文片段和产出的 wiki 页面，找出事实性错误。你的输出将写入报告供人工审核，**不会自动修改页面**。

## 检查方法

逐条阅读页面中的具体声明，在源文中找对应依据。重点：

1. **数字/日期**：页面中的数字是否与源文一致？有没有张冠李戴？
2. **归属**：页面说"A 做了 X"，源文是不是也这么说？有没有把 A 的事归到 B？
3. **因果/方向**：页面说"因为 A 所以 B"，源文是不是也这个方向？
4. **专有名词**：页面提到的人名、地名、术语、工具名，源文里有没有？
5. **遗漏导致歧义**：源文的限定条件被省略，导致页面的说法变成了不同的含义？

## 输出

没有错误：
```
PASS
```

有错误：
```
FAIL
- 页面写的: <引用页面原文，必须是页面中**逐字出现**的字符串>
  源文实际: <引用源文原文，必须是源文中**逐字出现**的字符串>
  源文行号: <源文中该原文所在的具体行号，如 line 797>
  错误类型: <数字错误/归属错误/方向反转/名词错误/遗漏致歧>
```

## 注意

- 只报事实性错误，不报风格/措辞问题
- 用自己的话概括源文是允许的，只要语义正确
- 省略次要细节是允许的，只要不改变核心含义
- 代码标识符（类名/函数名，出现在代码块或反引号内）与散文术语是两套词汇，不能互相替代。例如源文代码有 `PositionalEncoding`、散文有 "Positional Embedding 层"，两者都合法，不能因为代码写法不同就判页面用词错误
- 无法说出行号的疑似错误，宁可放过也不要凭印象指责
